# エラーハンドリングルール

## フロントエンド
- API呼び出しは必ず try/catch で囲む
- ユーザーにエラーメッセージを表示するUI（トースト等）を実装する
- ネットワークエラーと業務エラーを区別して表示する
- ローディング状態を実装する（スピナーやスケルトン）

## バックエンド
- 全APIエンドポイントにエラーハンドリングミドルウェアを設定する
- バリデーションエラーは 400 で返す
- 認証エラーは 401 で返す
- サーバーエラーは 500 で返し、スタックトレースは本番では隠す
- エラーレスポンスの形式を統一する:
  ```json
  { "error": { "code": "VALIDATION_ERROR", "message": "..." } }
  ```

## 禁止事項
- catch ブロックを空にしない
- console.error を本番コードに残さない（ロガーを使う）
- エラーを握りつぶさない（最低限ログに出す）
