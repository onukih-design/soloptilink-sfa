# 進捗管理ルール

## PLAN.md の更新タイミング
- 機能実装が完了したら、該当タスクの `[ ]` を `[x]` に変更する
- 新しいタスクが必要になったら、該当フェーズに追記する
- エラー修正に時間がかかった場合は、ログテーブルに記録する

## ログテーブルの書き方
```markdown
| 時刻 | 作業内容 | 結果 |
|------|----------|------|
| HH:MM | 〇〇機能を実装 | ✅ テスト通過 |
| HH:MM | □□のエラーを修正 | ✅ 3回目で解決 |
| HH:MM | △△のアプローチを変更 | ⚠️ 旧: X → 新: Y |
```

## コンテキスト管理
- /compact を実行したら、PLAN.md に「--- compact実行 ---」の区切りを入れる
- /clear を実行する前に、現在の作業内容をPLAN.mdのログに記録する
- 再開時は「docs/PLAN.md を読んで未完了タスクから再開して」で復帰できる
