# レビューパターン集（自動蓄積）

このファイルはチェーン開発のレビューセッションで発見された問題パターンを自動蓄積します。
新しいプロジェクトのレビュー時に、このファイルの知見がプロンプトに自動注入されます。

---

## 頻出パターン（初期データ）

### P001: エラーハンドリング不足
- 頻度: ★★★★★（ほぼ全プロジェクトで発生）
- 症状: try-catchがない、ユーザーにエラー内容が表示されない
- 影響: 本番で白い画面、ユーザー離脱
- 修正: 全API呼び出しにtry-catch、ローディング/エラー/空状態の3状態UI必須
- 確認コマンド: `grep -r "fetch\|axios" src/ | grep -v "catch\|try"`

### P002: バリデーション不足
- 頻度: ★★★★☆
- 症状: フォーム入力のバリデーションがない、SQLインジェクション可能
- 影響: セキュリティホール、データ破損
- 修正: フロントエンド（zod）+ バックエンド（zod）の二重バリデーション必須
- 確認コマンド: `grep -rL "z\.\|zod\|validate" src/api/ src/routes/`

### P003: レスポンシブ未対応
- 頻度: ★★★★☆
- 症状: PC画面だけで確認、モバイルで崩れる
- 影響: モバイルユーザーが使えない
- 修正: Tailwindのsm:/md:/lg:ブレークポイント確認、最低3画面幅でテスト
- 確認コマンド: `grep -r "sm:\|md:\|lg:" src/components/ | wc -l`

### P004: ローディング状態の欠如
- 頻度: ★★★★☆
- 症状: データ取得中に何も表示されない
- 影響: ユーザーが「壊れている」と判断して離脱
- 修正: 全非同期処理にisLoading状態 + Skeleton/Spinner表示
- 確認コマンド: `grep -r "isLoading\|loading\|Loading\|Spinner\|Skeleton" src/`

### P005: 環境変数のハードコード
- 頻度: ★★★☆☆
- 症状: APIキーやURLがソースコードに直書き
- 影響: セキュリティリスク、環境切替不可
- 修正: 全設定値を.envに移動、process.env.XXXで参照
- 確認コマンド: `grep -r "http://localhost\|127.0.0.1\|sk-\|api_key" src/`

### P006: N+1クエリ
- 頻度: ★★★☆☆
- 症状: ループ内でDB問い合わせ
- 影響: パフォーマンス劣化（10件→10回クエリ）
- 修正: JOINまたはIN句でバッチ取得
- 確認コマンド: `grep -rn "for\|forEach\|map" src/ | grep -i "find\|query\|select"`

### P007: テスト不足
- 頻度: ★★★★★
- 症状: テストファイルが少ない/空、エッジケース未テスト
- 影響: リグレッション、本番バグ
- 修正: 主要API全エンドポイント + 主要UIフローのE2Eテスト必須
- 確認コマンド: `find tests/ -name "*.test.*" | wc -l`

### P008: コスト最適化違反
- 頻度: ★★★☆☆（v3.1以降で検出開始）
- 症状: Claude/OpenAI APIの直接使用、有料DBの採用
- 影響: 月額コスト25倍
- 修正: DeepSeek/Geminiに切替、SQLite/Supabase Freeに切替
- 確認コマンド: `grep -ri "anthropic\|openai\|gpt-4\|claude-sonnet" src/`
