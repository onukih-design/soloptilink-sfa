# PMによる要件構築の具体例

## この文書の目的
「1行の思いつき」をPMに渡した場合に、どのような要件定義書が構築されるかの実例。

---

## インプット（ユーザーが渡したもの）

```
タスク管理ツール作って
```

**これだけ。1行。**

---

## PMの思考プロセス

### Phase 0: インプット理解
```
分類: D（1行の指示）
WHY:  チームのタスク進捗を可視化したい（推測）
WHO:  社内チーム 3-10名（推測）
WHAT: タスク管理Webアプリ
WHERE: Webブラウザ（PC + スマホ）
WHEN: 指定なし
HOW:  指定なし → React + TypeScript + Tailwind + Hono + SQLite を採用
```

### Phase 1: 競合調査
product-researcherが以下を調査:
- Trello → カンバンUI、Power-Up拡張
- Notion → 複数ビュー、データベース
- Asana → タイムライン、依存関係
- Todoist → シンプルUI、自然言語入力
- ClickUp → カスタムフィールド、ダッシュボード

結果:
| 機能 | Trello | Notion | Asana | Todoist | ClickUp | 判定 |
|------|--------|--------|-------|---------|---------|------|
| カンバンボード | ✅ | ✅ | ✅ | ❌ | ✅ | **MUST** |
| ステータス管理 | ✅ | ✅ | ✅ | ✅ | ✅ | **MUST** |
| 優先度 | ✅ | ✅ | ✅ | ✅ | ✅ | **MUST** |
| 期限設定+警告 | ✅ | ✅ | ✅ | ✅ | ✅ | **MUST** |
| フィルタリング | ✅ | ✅ | ✅ | ✅ | ✅ | **MUST** |
| 検索 | ✅ | ✅ | ✅ | ✅ | ✅ | **SHOULD** |
| ソート | ✅ | ✅ | ✅ | ❌ | ✅ | **SHOULD** |
| リスト/カンバン切替 | ❌ | ✅ | ✅ | ❌ | ✅ | **SHOULD** |
| コメント | ✅ | ✅ | ✅ | ✅ | ✅ | **SHOULD** |
| 一括操作 | ❌ | ✅ | ❌ | ❌ | ✅ | **SHOULD** |
| D&D並び替え | ✅ | ✅ | ✅ | ✅ | ✅ | **MUST** |
| ダッシュボード | ❌ | ❌ | ✅ | ❌ | ✅ | **SHOULD** |
| CSVエクスポート | ❌ | ✅ | ✅ | ❌ | ✅ | **NICE** |

### Phase 2: 要件定義書の構築（PMが自力で作成）

**出力: docs/FULL_REQUIREMENTS.md**

```
画面数:    7画面
機能数:    28機能
API数:     18エンドポイント
テーブル:  5テーブル
UX仕様:   12項目
```

---

## BEFORE → AFTER

| | ユーザーのインプット | PMが構築した要件定義書 |
|---|---|---|
| 文字数 | 10文字 | 約3,000文字 |
| 画面数 | 0 | **7画面** |
| 機能数 | 0 | **28機能** |
| API数 | 0 | **18エンドポイント** |
| テーブル数 | 0 | **5テーブル** |
| UX仕様 | 0 | **12項目** |

### PMが構築した画面一覧
1. ログイン画面
2. カンバンボード（メイン画面）
3. リストビュー
4. タスク詳細/編集モーダル
5. ダッシュボード
6. 設定画面
7. 404エラーページ

### PMが構築したテーブル
1. users（id, name, email, password_hash, role, created_at）
2. tasks（id, title, description, status, priority, assignee_id, due_date, position, column_id, created_at, updated_at）
3. columns（id, name, position, created_at）
4. comments（id, task_id, user_id, content, created_at）
5. activity_log（id, task_id, user_id, action, details, created_at）

### PMが構築したUX仕様
1. 検索: タスクタイトルでインクリメンタルサーチ（300msデバウンス）
2. フィルタ: ステータス、担当者、優先度、期限範囲
3. ソート: 作成日、期限、優先度、更新日（昇順/降順）
4. ショートカット: N=新規、Esc=閉じる、F=フィルタ開閉
5. D&D: カード移動（列内・列間）、列の並び替え
6. トースト: 操作成功時に右上3秒表示（緑=#10B981）
7. バリデーション: タイトル必須、期限は未来日のみ
8. ローディング: カンバン読み込み時にスケルトンスクリーン
9. 空状態: 「タスクはまだありません。Nキーで新規作成」
10. レスポンシブ: スマホではリスト表示のみ（カンバンは横スクロール）
11. 期限切れ: 赤バッジ表示 + ダッシュボードに一覧
12. 楽観的更新: D&D移動は即座に反映、失敗時にロールバック

**10文字のインプットから、実装可能な完全な仕様書が構築される。**
