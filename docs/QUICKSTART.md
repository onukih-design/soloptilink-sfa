# 3分クイックスタート

## Step 1: セットアップ（1分）

```bash
unzip claude-code-autonomous-dev-kit-v4.0.zip -d my-project
cd my-project
chmod +x setup.sh && ./setup.sh
```

## Step 2: チェーン開発で実行（推奨）

```bash
./chain.sh "タスク管理ツール作って"
```

これだけで以下が全自動で実行されます:
1. **開発セッション**: PMが要件構築→100体エージェントで実装→テスト通過
2. **レビューセッション**: フレッシュな目でコード全体を10観点でスコアリング
3. **修正セッション**: レビュー指示に基づいて全修正
4. **再レビュー**: 品質スコアが閾値(85/100)を超えるまで繰り返し

### ラウンド数を増やす
```bash
./chain.sh "AIチャットボット付きFAQ" --rounds 3
```

## Step 2b: 従来の単一セッション開発

```bash
claude
# → @.claude/agents/pm.md タスク管理ツール作って。全部やって。止まらないで。
```

## 便利なコマンド

```bash
./chain.sh --review-only    # 既存コードのレビューだけ
./chain.sh --fix-only       # レビューに基づく修正だけ
./chain.sh --continue       # 前回の続きから
```

## なぜチェーン開発か？

| 方式 | 品質 | 理由 |
|------|------|------|
| 単一セッション | ★★★☆☆ | 長いセッションでコンテキストが汚れる |
| チェーン（1ラウンド） | ★★★★☆ | レビューで問題を発見→修正 |
| チェーン（2ラウンド） | ★★★★★ | 2回のレビューで見落としも修正 |

**1セッション目が最も品質が高い。だから全てを別セッションで行う。**

## うまくいかない場合

| 状況 | 対処 |
|------|------|
| 途中で止まった | `./chain.sh --continue` |
| レビュースコアが低い | `./chain.sh --continue` でさらにラウンド追加 |
| 意図と違うものができた | より具体的に伝えて `./chain.sh` 再実行 |
