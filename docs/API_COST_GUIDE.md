## 設計原則: AI階層ルーティング

### 原則1: 最安モデルから試す
```
ユーザーリクエスト
  ↓
Gemini Flash-Lite（$0.10/$0.40）で分類
  ├─ 簡単なタスク → Gemini Flash で処理（$0.10/$0.40）
  ├─ 中程度のタスク → DeepSeek V3.2 で処理（$0.28/$0.42）
  └─ 複雑なタスク → DeepSeek V3.2 Reasoner で処理（$0.28/$0.42）
      └─ 品質不足 → Gemini 2.5 Pro にフォールバック（$1.25/$10）
```

### 原則2: キャッシュを最大活用する
- DeepSeek: 自動キャッシュ。同じプレフィックスで90%コスト削減
- Gemini: Context Cachingで入力コスト75%削減
- アプリ側: Redis/メモリキャッシュでAPI呼び出し自体を削減

### 原則3: バッチ処理で半額
- 即時性不要なタスク → Batch APIで50%OFF
- 例: 日次レポート生成、データ分析、コンテンツ生成

### 原則4: OpenAI互換APIで柔軟に切替
- DeepSeekはOpenAI互換フォーマット
- base_urlとapi_keyの変更だけでプロバイダ切替可能
- 将来のさらに安いモデルへの移行が容易

## コスト設計チェックリスト

PMおよびarchitectは、設計時に以下を全て確認する:

- [ ] AI機能にClaude APIを直接使っていないか？（DeepSeek/Geminiを優先）
- [ ] AI呼び出しにキャッシュ層があるか？
- [ ] バッチ処理可能なタスクをリアルタイムで処理していないか？
- [ ] AIルーティング（安いモデル→高いモデル）が設計されているか？
- [ ] DBはSQLite/Tursoなど最安オプションを検討したか？
- [ ] ホスティングは無料枠を最大活用しているか？
- [ ] 認証は無料サービスを使っているか？
- [ ] 外部APIは無料枠を最大活用しているか？
- [ ] 月額ランニングコストの見積もりがdocs/COST_ESTIMATE.mdに記載されているか？
