## 第1層: 機能面のギャップ（競合調査ベース）

競合調査（Salesforce / HubSpot / Mazrica Sales / GENIEE / kintone）の結果、要件定義書に明記されていなかった機能を以下に整理する。

### MUST追加機能（5項目）

競合水準で必須と判断された機能。未実装の場合、ユーザーの期待値を下回るリスクがある。

| # | 機能 | 要件定義書の状態 | ギャップ内容 | 対応Phase |
|---|------|----------------|-------------|----------|
| 1 | ダッシュボードKPIカード | **部分的に記載あり** — F8（売上ダッシュボード）にMRR・グラフの記載はあるが、「本日のアクション数」「パイプライン金額サマリー」等のKPIカード単体の定義は不十分 | KPIカードの具体的な項目・レイアウトを定義に追加 | Phase 1 |
| 2 | 活動ログ/ステータス変更履歴 | **部分的に記載あり** — F2（案件詳細）に「ステータス遷移の履歴ログ表示」、F3に「ヨミステータス変更時の自動ログ記録」の記載あり。ただし、ログのデータ構造（テーブル設計）や表示UIが未定義 | activity_logs テーブルの追加設計、履歴一覧UIの定義が必要 | Phase 1 |
| 3 | トースト通知/操作フィードバック | **実装ルールにのみ記載** — 11.4「エラーハンドリングはtry-catchで必ず行い、ユーザー向けトースト通知で表示する」と記載あるが、成功時・警告時のフィードバック仕様が未定義 | 成功/エラー/警告の3パターンのトースト仕様を機能要件に追加 | Phase 1 |
| 4 | バルク操作（一括編集/一括削除） | **記載なし** — 案件管理表は個別セル編集のみの定義 | チェックボックス選択 + 一括ステータス変更/一括クローザー変更/一括削除の機能追加 | Phase 2 |
| 5 | データバリデーション | **記載なし** — F14（Excelインポート）に「バリデーション＋エラーレポート表示」の記載はあるが、通常の入力フォームにおけるバリデーション仕様が未定義 | 必須項目チェック、型チェック、値範囲チェック、重複チェックのルール定義が必要 | Phase 1 |

### SHOULD追加機能（7項目）

品質向上・差別化のために追加すべき機能。

| # | 機能 | 要件定義書の状態 | ギャップ内容 | 対応Phase |
|---|------|----------------|-------------|----------|
| 1 | キーボードショートカット | **部分的に記載あり** — 6.2に「Tab/矢印キーによるセル間移動」「Ctrl+C/V」の記載あり。ただしショートカット一覧の体系的定義なし | ショートカットキー一覧表の作成、Cmd/Ctrl+S（保存）等の追加定義 | Phase 1 |
| 2 | 列の表示/非表示切替 | **記載あり** — 6.2に「列の表示/非表示切替」の記載あり | ギャップなし（実装時のUI仕様詳細化が必要） | Phase 1 |
| 3 | 列幅のドラッグ調整 | **記載あり** — 6.2に「列幅のドラッグ調整」の記載あり | ギャップなし（実装時のUI仕様詳細化が必要） | Phase 1 |
| 4 | 空状態（Empty State）表示 | **記載なし** — データが0件の画面に対するUI仕様が未定義 | 各画面の空状態UIデザイン・メッセージ定義が必要（第2層で詳述） | Phase 1 |
| 5 | スケルトンローディング | **記載なし** — ローディング中のUI仕様が未定義 | 各画面のローディング表示パターン定義が必要（第2層で詳述） | Phase 1 |
| 6 | フリーワード検索（グローバル） | **部分的に記載あり** — F4に「フリーワード検索」の記載あり。ただし検索対象範囲が「案件一覧」に限定されており、全画面横断の定義なし | グローバルヘッダーに常設する横断検索バーの定義が必要 | Phase 2 |
| 7 | CSV/PDFエクスポート | **記載なし** — F15はExcel形式のみ | CSV/PDFの出力形式追加 | Phase 4 |

## 第2層: 技術面のギャップ

要件定義書に明示されていないが、プロダクション品質のWebアプリケーションとして技術的に必須の項目を以下に整理する。

### TG-1: エラーUI（トースト通知・バリデーションメッセージ・エラーページ）

| 項目 | 要件定義書の状態 | 必要な定義 |
|------|----------------|-----------|
| トースト通知 | 11.4にエラー時のトースト記載のみ | 成功（緑）/ エラー（赤）/ 警告（黄）/ 情報（青）の4種類、表示位置（右上）、自動消去時間（5秒）、手動閉じボタン |
| バリデーションメッセージ | 記載なし | フィールド直下への赤文字エラー表示、リアルタイムバリデーション（onBlur）、送信時の一括バリデーション |
| 404ページ | 記載なし | 存在しないURLアクセス時の案内ページ（ダッシュボードへのリンク付き） |
| 500ページ | 記載なし | サーバーエラー時の案内ページ（再試行ボタン付き） |

### TG-2: ローディング表示

| 項目 | 要件定義書の状態 | 必要な定義 |
|------|----------------|-----------|
| スピナー | 記載なし | ボタンクリック後の処理中スピナー（ボタン内インライン表示） |
| スケルトンスクリーン | 記載なし | テーブル/カード/グラフの読込中にプレースホルダー表示 |
| ページ遷移ローディング | 記載なし | Next.js App Routerの`loading.tsx`による画面遷移中の表示 |
| 初期ロード | 非機能要件に「2秒以内」の記載あり | 2秒以内に表示されない場合のプログレス表示 |

### TG-3: 空状態（Empty State）表示

| 画面 | 必要な空状態メッセージ例 |
|------|----------------------|
| 案件一覧 | 「案件がまだ登録されていません。最初の案件を登録しましょう」+ 新規登録ボタン |
| 受注管理表 | 「受注データがありません」 |
| ダッシュボード | 「データが不足しています。案件を登録するとKPIが表示されます」 |
| 分析レポート | 「分析に必要なデータがまだ蓄積されていません」 |
| 検索結果0件 | 「条件に一致する案件が見つかりませんでした」+ フィルタリセットボタン |

### TG-4: 認証フロー

| 項目 | 要件定義書の状態 | 必要な定義 |
|------|----------------|-----------|
| ログインページ | 11.2に`(auth)/`の記載あり | メール/パスワード入力フォーム、バリデーション、エラー表示（「メールアドレスまたはパスワードが正しくありません」） |
| ログイン失敗 | 記載なし | 失敗回数のカウント、連続失敗時の注意表示 |
| セッション切れ | 記載なし | Supabase Authのトークンリフレッシュ失敗時にログイン画面へリダイレクト、「セッションが切れました。再度ログインしてください」メッセージ表示 |
| 認証リダイレクト | 記載なし | 未認証状態で保護ページにアクセスした場合のログインページへのリダイレクト、ログイン後の元ページへの復帰 |
| ログアウト | 記載なし | ログアウト確認、セッション破棄、ログインページへのリダイレクト |

### TG-5: レスポンシブ対応

| 項目 | 要件定義書の状態 | 必要な定義 |
|------|----------------|-----------|
| ブレークポイント | 非機能要件に「PC（メイン）、タブレット対応。スマートフォンは閲覧のみ」の記載あり | 具体的なブレークポイント定義: sm(640px) / md(768px) / lg(1024px) / xl(1280px) |
| スマートフォン表示 | 「閲覧のみ対応」の記載あり | テーブルの横スクロール、ハンバーガーメニュー、カード形式への切替など具体的なUIパターンが未定義 |
| タブレット表示 | 「タブレット対応」の記載あり | サイドバーの折りたたみ、テーブル表示の調整など具体的な挙動が未定義 |
| ナビゲーション | 記載なし | PC: 固定サイドバー / タブレット: 折りたたみサイドバー / スマホ: ハンバーガーメニュー |

### TG-6: エラーページ

| ページ | 必要な定義 |
|--------|-----------|
| 404 Not Found | Next.js `not-found.tsx` — イラスト/アイコン + 「ページが見つかりません」+ ダッシュボードへ戻るボタン |
| 500 Internal Server Error | Next.js `error.tsx` — 「サーバーエラーが発生しました」+ 再試行ボタン + 問題が続く場合の案内 |
| 認証エラー | 未認証/権限不足時の403相当ページ — 「アクセス権限がありません」+ ログインページへのリンク |

### TG-7: メタ情報

| 項目 | 要件定義書の状態 | 必要な定義 |

| ページタイトル | 記載なし | 各ページの`<title>`タグ: 例「案件一覧 | SoloptiLink」 |
| ファビコン | 記載なし | `/public/favicon.ico` — SoloptiLinkのブランドアイコン |
| OGP/メタディスクリプション | 記載なし | 社内ツールのため最低限（Slack等でURLシェア時にタイトルが表示される程度） |

## 次のアクション

上記ギャップを要件定義書v1.2として反映し、以下の順序で補完することを推奨する。

1. **即時対応（開発開始前）**: TG-8（環境変数）、TG-9（ビルド設定）
2. **Phase 1着手時**: TG-1（エラーUI）、TG-2（ローディング）、TG-4（認証フロー）、MUST #4（バルク操作）、MUST #5（バリデーション）
3. **Phase 1完了まで**: TG-3（空状態）、TG-5（レスポンシブ詳細）、TG-6（エラーページ）、TG-7（メタ情報）
4. **Phase 4**: SHOULD #7（CSV/PDFエクスポート）
